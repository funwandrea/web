Data Analysis and Modeling
Multiple Regression analysis were run to find the optimal outcome. Turns out if you just fill the missing data with median yielded better results.
Gradient Boosting Classifier, XGBoost(make sure you run it), and Logistic Regression were used to analyzed the data.

Since the company wanted to have less false positives. I put more consideration on Precision and recall vs the accuracy of the result.
When missing data was filled with the median for credit score Gradient Boosting classifier yielded 98% precise and recall. When using K-Nearest Neighbors to fill the missing values for credit score it yielded a lower precision and recall score but a higher accuracy score
Results and Proposal
Gradient boosting classifier algorithm was used to determine pre-qualification status. Using a more robust algorithm like xgboost yielded slightly better results but would take more resources and the credit union wanted to use the least amount of resources.

Top ten variables were shown to be weighing the heaviest on the algorithm and a new algorithm was built based off these variables. A basic user interface was created where loan officers or customers can type in their information online and get an answer quickly if the company will qualify them for a loan. The credit union was given the predictive model as well as the ten variables that contributed to the algorithm the most.
